# Feature: Read

> 상위 문서: [[../Plan]]

## 목적

`data.json`에 저장된 레코드를 조회한다. 두 가지 하위 기능을 제공한다.

1. 전체 목록 보기
2. id 또는 특정 필드 값으로 검색

## 관련 API

- `PersonStore::all() -> const json::Array&` — 전체 목록
- `PersonStore::findById(int id) -> const json::Value*` — id 단건 조회
- `PersonStore::findByField(field, value) -> std::vector<const json::Value*>` — 필드 값 검색

## 사용자 흐름

```
-- Read --
1. 전체 목록 보기
2. id로 검색
3. 필드 값으로 검색
번호 선택 >
```

### 1) 전체 목록 보기

```
[id=1] 홍길동 / 30세 / 학생 아님 / 기술: C++, Python
[id=2] 김철수 / 22세 / 학생 / 기술: (없음)
총 2건
```

- 레코드가 없는 경우: `저장된 데이터가 없습니다.` 출력

### 2) id로 검색

```
검색할 id 입력: 1

{
  "id": 1,
  "name": "홍길동",
  "age": 30,
  "isStudent": false,
  "skills": ["C++", "Python"]
}
```

- 존재하지 않는 id: `id=1 에 해당하는 데이터가 없습니다.` 출력 (에러로 취급하지 않음)

### 3) 필드 값으로 검색

```
검색할 필드 이름 (name/age/isStudent) 입력: name
검색어 입력: 길동

검색 결과 1건:
[id=1] 홍길동 / 30세 / 학생 아님 / 기술: C++, Python
```

- 검색 대상 필드는 `name`, `age`, `isStudent`로 제한한다 (skills 배열 검색은 범위 밖).
- `name`: 부분 문자열 포함 여부로 매칭 (대소문자 구분 없음)
- `age`: 정확히 일치하는 값만 매칭 (문자열을 정수로 변환 후 비교, 변환 실패 시 "숫자를 입력하세요" 안내)
- `isStudent`: `y`/`n` 입력을 bool로 변환 후 정확히 일치하는 값만 매칭
- 지원하지 않는 필드명 입력 시: "지원하지 않는 검색 필드입니다 (name/age/isStudent 중 선택)" 안내 후 재입력

## 출력 형식

- 목록/검색 결과는 사람이 읽기 쉬운 요약 한 줄 형식(`[id=..] 이름 / 나이 / 학생여부 / 기술`)을 기본으로 하고,
  id 단건 조회에는 `dump(2)`로 원본 JSON 구조도 함께 보여준다 (구조 확인 목적).

## 에지 케이스

- 검색 결과가 0건인 경우 오류가 아니라 "검색 결과가 없습니다."로 안내
- `data.json`이 비어있는 상태에서 검색 시도: "저장된 데이터가 없습니다." 안내 후 메뉴로 복귀
- skills가 빈 배열인 레코드의 출력: `기술: (없음)`으로 표기

## 테스트 체크리스트

- [ ] 레코드 0건일 때 전체 목록 보기가 "저장된 데이터가 없습니다."를 출력하는지
- [ ] 존재하는 id 검색 시 해당 레코드가 정확히 출력되는지
- [ ] 존재하지 않는 id 검색 시 에러 없이 안내 메시지만 출력되는지
- [ ] name 필드 부분 일치 검색(대소문자 무관)이 동작하는지
- [ ] age 필드 정확 일치 검색이 동작하는지
- [ ] isStudent 필드 y/n 검색이 동작하는지
- [ ] 지원하지 않는 필드명 입력 시 재입력을 요청하는지
