# Feature: Create

> 상위 문서: [[../Plan]]

## 목적

사용자로부터 새 인물 데이터를 입력받아 검증 후 `data.json`에 새 레코드로 추가한다.

## 관련 API

`PersonStore::addPerson(name, age, isStudent, skills) -> int (새로 부여된 id)`

## 사용자 흐름

```
-- Create --
이름 입력: 홍길동
나이 입력: 30
학생 여부 (y/n): n
보유 기술 입력 (콤마로 구분, 없으면 Enter): C++,Python

생성 완료 (id=3)
{
  "id": 3,
  "name": "홍길동",
  "age": 30,
  "isStudent": false,
  "skills": ["C++", "Python"]
}
```

## 입력 항목 및 검증 규칙

| 항목 | 검증 규칙 | 실패 시 동작 |
|---|---|---|
| 이름 | 공백을 제외한 문자가 1개 이상 | "이름은 비워둘 수 없습니다" 출력 후 재입력 요청 |
| 나이 | 정수, 0 이상 (상한은 두지 않되 비정상적으로 큰 값은 허용) | "나이는 0 이상의 정수여야 합니다" 출력 후 재입력 요청 |
| 학생 여부 | `y`/`n`(대소문자 무관)만 허용 | "y 또는 n을 입력하세요" 출력 후 재입력 요청 |
| 보유 기술 | 콤마로 분리, 각 항목의 앞뒤 공백 제거, 빈 입력 시 빈 배열로 저장 | 검증 실패 케이스 없음 (빈 항목은 무시하고 필터링) |

- `id`는 사용자에게 입력받지 않는다. `PersonStore::nextId()`가 자동으로 부여한다.
- 각 재입력 요청은 무한 루프가 아니라 최대 시도 없이 유효한 값이 들어올 때까지 반복한다
  (Ctrl+C/EOF로만 중단 가능 — 별도 취소 명령은 두지 않는다).

## 저장 절차

1. 입력값 검증 통과
2. `PersonStore::addPerson(...)` 호출 → 내부적으로 `nextId()`로 id 결정, 레코드를 메모리 배열 끝에 append
3. `PersonStore::save()` 호출로 `data.json` 즉시 갱신
4. 방금 추가된 레코드를 화면에 pretty-print(`dump(2)`)로 보여줘 사용자가 결과를 확인할 수 있게 함

## 에지 케이스

- `data.json` 파일이 아직 없는 최초 실행: `PersonStore::load()`가 빈 배열로 시작하므로 첫 Create 시 id=1 부여
- 이름에 쉼표(`,`)가 포함된 경우: 이름 필드는 통째로 한 줄 입력을 받으므로 문제 없음 (콤마 분리는 skills에만 적용)
- 매우 큰 age 값(예: overflow 근처): `std::stoi` 예외 발생 시 "숫자 형식이 올바르지 않습니다"로 안내 후 재입력

## 테스트 체크리스트

- [ ] 정상 입력으로 레코드가 추가되고 id가 1부터 순차 부여되는지
- [ ] 빈 이름 입력 시 재입력을 요청하는지
- [ ] 나이에 문자열("abc") 입력 시 재입력을 요청하는지
- [ ] 학생 여부에 y/n 이외 값 입력 시 재입력을 요청하는지
- [ ] skills를 빈 입력으로 두면 `[]`로 저장되는지
- [ ] 프로그램 재시작 후에도 이전에 Create한 레코드가 `data.json`에서 로드되는지
